### Step 1: 确保窗口在前台

**操作：**
```python
wx.bring_to_front()
```

**实现逻辑：**
```python
def bring_to_front(self):
    hwnd = self.find_window()  # FindWindowW("WeWorkWindow", None)
    foreground = user32.GetForegroundWindow()
    if foreground == hwnd:
        return  # 已在前台，无需操作
    if not user32.IsWindowVisible(hwnd):
        user32.ShowWindow(hwnd, 9)  # SW_RESTORE
    user32.SetForegroundWindow(hwnd)
    time.sleep(self.action_delay)  # 默认0.3秒
```

**检测点：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 窗口存在 | `FindWindowW` 返回非0句柄 | 抛出异常 "企业微信窗口未找到" |
| 窗口可见 | `IsWindowVisible` 返回True | 调用 `ShowWindow` 恢复窗口 |

**实现逻辑：**
```python
def type_text(self, text, delay=0.02):
    for char in text:
        # SendInput 模拟键盘输入（支持中文）
        inp_down = INPUT()
        inp_down.type = INPUT_KEYBOARD
        inp_down.union.ki.wScan = ord(char)
        inp_down.union.ki.dwFlags = KEYEVENTF_UNICODE
        user32.SendInput(1, ctypes.pointer(inp_down), ctypes.sizeof(INPUT))
        time.sleep(delay)
        # ... 释放按键
```

**检测点：** 无（输入操作本身不校验）

**等待：** 2.0秒（等待搜索结果出现）

### Step 9: 查找"添加"按钮（关键步骤）

**操作：**
```python
# 三种方法按优先级尝试
add_item = None

# 方法1: 右侧阈值
for item in items:
    if "添加" in item["text"]:
        if item["window_pos"][0] > max(200, window_w * 0.35):
            add_item = item
            break

# 方法2: 相对位置
if not add_item and lookup_item:
    lx, ly = lookup_item["window_pos"]
    for item in items:
        if "添加" in item["text"]:
            ax, ay = item["window_pos"]
            if ax > lx + 50 and abs(ay - ly) < 80:
                add_item = item
                break

# 方法3: 兜底
if not add_item:
    for item in items:
        if "添加" in item["text"]:
            add_item = item
            break
```

**三种方法对比：**

| 方法 | 优先级 | 逻辑 | 适用场景 |
|------|--------|------|----------|
| 方法1 | 高 | x > max(200, window_w * 0.35) | 窗口位置固定，左侧边栏约200px |
| 方法2 | 中 | x > lx + 50 且 \|y - ly\| < 80 | "添加"在"网络查找"右侧同一水平线 |
| 方法3 | 低 | 只要找到"添加"就用 | 兜底方案，不限制位置 |

**检测点：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 添加按钮存在 | `add_item is not None` | 按Escape，返回失败 |

### Step 11: 验证弹窗"发送添加邀请"出现

**操作：**
```python
if not wx.verify_action_result("发送添加邀请", timeout=3, should_exist=True):
    return {"status": "failed", "detail": "点击添加后未出现弹窗"}
```

**实现逻辑：**
```python
def verify_action_result(self, expected_text, timeout=3, should_exist=True):
    start = time.time()
    while time.time() - start < timeout:
        found = self.find_text(expected_text)
        if should_exist and found:
            return True  # 应该存在且找到了
        if not should_exist and not found:
            return True  # 应该消失且未找到
        time.sleep(0.3)
    return False
```

**检测点：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 弹窗出现 | 3秒内找到"发送添加邀请" | 按Escape，返回失败 |

### Step 14: 查找"发送"按钮（关键步骤）

**操作：**
```python
# 1. 找到弹窗标题位置
popup_title = wx.find_text("发送添加邀请")
tx, ty = popup_title["window_pos"]

# 2. 筛选弹窗内的"发送"（标题下方50-250px）
popup_send_items = []
for item in send_items:
    x, y = item["window_pos"]
    if ty + 50 < y < ty + 250:
        popup_send_items.append(item)

# 3. 选择最靠近标题的（y最小的）
if popup_send_items:
    popup_send_items.sort(key=lambda x: x["window_pos"][1])
    send_btn = popup_send_items[0]
else:
    # 兜底：使用原逻辑
    valid_items = [item for item in send_items if item["window_pos"][1] > ty + 30]
    valid_items.sort(key=lambda x: x["window_pos"][1])
    send_btn = valid_items[-1]
```

**关键逻辑说明：**

| 场景 | y坐标范围 | 处理 |
|------|-----------|------|
| "发送添加邀请"标题 | y ≈ ty | 排除（在上方） |
| 弹窗"发送"按钮 | ty + 50 < y < ty + 250 | **选中** |
| 群聊天框"发送" | y > ty + 250（通常在底部） | 排除（在下方远处） |

**检测点：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 弹窗标题存在 | `popup_title is not None` | 返回失败 |
| 发送按钮存在 | `len(send_items) >= 1` | 返回失败 |
| 有效发送按钮 | 筛选后有候选或兜底成功 | 返回失败 |

## 关键设计决策

### 1. 为什么使用多种方法定位"添加"按钮？

**问题：** 企业微信界面可能有不同的布局
- 窗口大小不同
- DPI设置不同
- 弹窗位置可能变化

**解决方案：** 三种方法+兜底方案，提高兼容性

### 2. 为什么"发送"按钮要基于标题相对位置？

**问题：** 弹窗内"发送"按钮和群聊天框"发送"按钮都被识别到

**原方案（有问题）：**
- 按y坐标排序选择最大的
- 群聊天框的"发送"在底部（y更大），会被选中

**新方案：**
- 基于弹窗标题"发送添加邀请"的相对位置
- 只选择标题下方50-250px范围内的"发送"
- 群聊天框的"发送"在底部（y > 标题y + 250），被排除

### 3. 为什么输入验证仅警告不阻断？

**问题：** SendInput模拟键盘输入可能失败
- 窗口失去焦点
- 输入法问题
- 系统拦截

**解决方案：** OCR验证可以检测输入是否成功，但仅警告不阻断，避免过度严格。

### 4. 为什么弹窗关闭验证仅警告不返回失败？

**问题：** 发送操作可能已经成功了
- 弹窗关闭有延迟
- OCR扫描有延迟
- 企业微信动画效果

**解决方案：** 仅警告可以避免误判成功为失败。
