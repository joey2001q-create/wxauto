# 企业微信添加联系人流程 - 详细步骤与校验逻辑

## 流程概览

```
开始
  ↓
[Step 1] 确保窗口在前台
  ↓
[Step 2] 点击"消息"标签
  ↓
[Step 3] 点击搜索框
  ↓
[Step 4] 输入手机号
  ↓
[Step 5] 查找"网络查找"
  ↓ 失败 → 返回失败
  ↓
[Step 6] 点击"网络查找"
  ↓
[Step 7] 检查"用户不存在"
  ↓ 存在 → 返回"phone_not_exist"
  ↓
[Step 8] 检查是否已是好友("发消息")
  ↓ 存在 → 返回"already_friend"
  ↓
[Step 9] 查找"添加"按钮
  ↓ 失败 → 返回失败
  ↓
[Step 10] 点击"添加"按钮
  ↓
[Step 11] 验证弹窗"发送添加邀请"出现
  ↓ 失败 → 返回失败
  ↓
[Step 12] 定位输入框
  ↓ 失败 → 返回失败
  ↓
[Step 13] 输入验证消息
  ↓
[Step 14] 查找"发送"按钮
  ↓ 失败 → 返回失败
  ↓
[Step 15] 点击"发送"按钮
  ↓
[Step 16] 验证弹窗关闭
  ↓
[Step 17] 关闭搜索
  ↓
返回"success"
```

## 详细步骤说明

### Step 1: 确保窗口在前台

**操作：**
```python
wx.bring_to_front()
```

**实现逻辑：**
1. 查找窗口句柄 `FindWindowW("WeWorkWindow", None)`
2. 获取当前前台窗口 `GetForegroundWindow()`
3. 如果窗口不是前台：
   - 如果窗口不可见，恢复窗口 `ShowWindow(hwnd, 9)`
   - 设置前台窗口 `SetForegroundWindow(hwnd)`
4. 等待 `action_delay` 秒（默认0.3秒）

**校验：** 无（失败会抛出异常）

---

### Step 2: 点击"消息"标签

**操作：**
```python
wx.click_text("消息")
```

**实现逻辑：**
1. OCR扫描查找"消息"文字
2. 点击找到的坐标

**校验：**
- 如果找不到"消息"，捕获异常继续（可能已在消息页面）

**失败处理：** 继续执行

---

### Step 3: 点击搜索框

**操作：**
```python
search_item = wx.find_text("搜索")
wx.click(search_item["screen_pos"])
```

**实现逻辑：**
1. OCR扫描查找"搜索"文字
2. 点击搜索框坐标

**校验：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 搜索框存在 | `search_item is not None` | 返回失败"未找到搜索框" |

**等待：** 0.5秒

---

### Step 4: 输入手机号

**操作：**
```python
wx.ctrl_a()          # 全选
wx.press_backspace() # 删除
wx.type_text(phone)  # 输入手机号
```

**实现逻辑：**
1. Ctrl+A 全选原有内容
2. Backspace 删除
3. 输入手机号（使用SendInput模拟键盘）

**校验：** 无（输入操作本身不校验）

**等待：** 2.0秒（等待搜索结果出现）

---

### Step 5: 查找"网络查找"

**操作：**
```python
lookup_item = wx.find_text("网络查找")
```

**实现逻辑：**
1. 优先查找"网络查找"文字
2. 如果找不到，查找手机号本身（纯数字匹配）

**校验：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 搜索结果存在 | `lookup_item is not None` | 按Escape关闭搜索，返回"not_found" |

---

### Step 6: 点击"网络查找"

**操作：**
```python
wx.click(lookup_item["screen_pos"])
```

**实现逻辑：**
1. 点击"网络查找"或手机号的坐标

**校验：** 无

**等待：** 1.5秒（等待详情页面加载）

---

### Step 7: 检查"用户不存在"

**操作：**
```python
hints = ["该用户不存在", "无法找到该用户", "用户不存在"]
for hint in hints:
    if wx.find_text(hint):
        # 用户不存在
```

**实现逻辑：**
1. 检查是否存在"用户不存在"相关提示
2. 如果存在，点击"确定"关闭提示

**校验：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 用户是否存在 | 未找到不存在提示 | 继续执行 |
| | 找到不存在提示 | 按Escape，返回"phone_not_exist" |

---

### Step 8: 检查是否已是好友

**操作：**
```python
wx.find_text_safe("发消息", confirm_times=1, require_stable=False)
```

**实现逻辑：**
1. 使用安全查找检测"发消息"按钮
2. 只需找到1次，不要求位置稳定

**校验：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 是否已是好友 | 未找到"发消息" | 继续执行 |
| | 找到"发消息" | 按Escape，返回"already_friend" |

---

### Step 9: 查找"添加"按钮

**操作：**
```python
# 方法1: 右侧阈值
for item in items:
    if "添加" in item["text"]:
        if item["window_pos"][0] > max(200, window_w * 0.35):
            add_item = item
            break

# 方法2: 兜底方案
if not add_item:
    for item in items:
        if "添加" in item["text"]:
            add_item = item
            break
```

**实现逻辑（三种方法）：**

| 方法 | 优先级 | 逻辑 | 适用场景 |
|------|--------|------|----------|
| 方法1 | 高 | x > max(200, window_w * 0.35) | 窗口位置固定 |
| 方法2 | 中 | 基于"网络查找"相对位置 | 弹窗位置变化 |
| 方法3 | 低 | 只要找到"添加"就用 | 兜底方案 |

**校验：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 添加按钮存在 | `add_item is not None` | 按Escape，返回失败"未找到'添加'按钮" |

---

### Step 10: 点击"添加"按钮

**操作：**
```python
wx.click(add_item["screen_pos"])
```

**实现逻辑：**
1. 点击"添加"按钮的屏幕坐标

**校验：** 无

**等待：** 1秒（等待弹窗出现）

---

### Step 11: 验证弹窗"发送添加邀请"出现

**操作：**
```python
wx.verify_action_result("发送添加邀请", timeout=3, should_exist=True)
```

**实现逻辑：**
1. 在3秒内轮询检查"发送添加邀请"文字
2. 间隔0.3秒检查一次

**校验：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 弹窗是否出现 | 3秒内找到"发送添加邀请" | 继续执行 |
| | 3秒内未找到 | 按Escape，返回失败"点击添加后未出现弹窗" |

---

### Step 12: 定位输入框

**操作：**
```python
input_x, input_y, method = wx.locate_input_box_multi_check(phone, verify_msg)
wx.click(input_x, input_y)
```

**实现逻辑（多重检查机制）：**

| 方法 | 优先级 | 逻辑 |
|------|--------|------|
| 方法1 | 高 | 轮廓检测（基于标题ROI找输入框） |
| 方法2 | 中 | OpenCV找叉号图标 |
| 方法3 | 低 | OCR+相对坐标（从"发送"按钮推算） |

**交叉验证：**
- 如果多个方法都成功，计算结果间距离
- 距离 < 50像素：取平均值
- 距离 >= 50像素：选择置信度高的结果

**校验：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 输入框定位成功 | 至少一种方法成功 | 继续执行 |
| | 所有方法失败 | 按Escape，返回失败"定位输入框失败" |

**等待：** 0.3秒

---

### Step 13: 输入验证消息

**操作：**

**实现逻辑：**
1. Ctrl+A 全选原有内容
2. Backspace 删除
3. 输入验证消息

**校验（可选）：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 输入是否成功 | OCR检查是否包含验证消息 | 仅警告，继续执行 |

**等待：** 0.3秒

### Step 14: 查找"发送"按钮

**操作：**
```python
send_items = wx.find_all_text("发送")
```

**实现逻辑：**
1. 查找所有包含"发送"的文字
2. 按y坐标排序，选择最下方的（y最大的）

**关键逻辑：**
```python
if len(send_items) >= 2:
    # 按y坐标排序，找最下方的
    send_items.sort(key=lambda x: x["window_pos"][1])
    send_btn = send_items[-1]
elif len(send_items) == 1:
    send_btn = send_items[0]
    # 检查y坐标是否在下方（> 300）
    if send_btn["window_pos"][1] <= 300:
        # 可能是标题而非按钮
```

**校验：**
| 检查项 | 条件 | 失败处理 |
|--------|------|----------|
| 发送按钮存在 | `len(send_items) >= 1` | 返回失败"未找到发送按钮" |
| 发送按钮位置 | y > 300（如果只有一个） | 返回失败"找到的发送文字位置异常" |

## 返回状态码

| 状态码 | 含义 | 触发条件 |
|--------|------|----------|
| `success` | 成功 | 发送邀请成功 |
| `already_friend` | 已是好友 | 找到"发消息"按钮 |
| `not_found` | 未找到 | 未找到"网络查找" |
| `phone_not_exist` | 手机号不存在 | 找到"用户不存在"提示 |
| `failed` | 失败 | 其他各种失败情况 |

## 关键设计决策

### 1. 为什么使用多种方法定位"添加"按钮？

企业微信界面可能有不同的布局：
- 窗口大小不同
- DPI设置不同
- 弹窗位置可能变化

使用多种方法+兜底方案提高兼容性。

### 2. 为什么"发送"按钮要按y坐标排序？

弹窗内有两个"发送"相关文字：
- 标题"发送添加邀请"（在上方）
- 按钮"发送"（在下方）

按y坐标排序选择最下方的，确保点击的是按钮而非标题。

### 3. 为什么输入验证消息后要做OCR验证？

因为SendInput模拟键盘输入可能失败：
- 窗口失去焦点
- 输入法问题
- 系统拦截

OCR验证可以检测输入是否成功，但仅警告不阻断，避免过度严格。

### 4. 为什么弹窗关闭验证仅警告不返回失败？

因为发送操作可能已经成功了：
- 弹窗关闭有延迟
- OCR扫描有延迟
- 企业微信动画效果

仅警告可以避免误判成功为失败。
